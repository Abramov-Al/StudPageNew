Лабораторная работа 2
Перевод кода JSON в диаграмму UML Class

Описание основных задач

1. Требования к программной части проекта:

  • Проект должен быть реализован в виде одной динамической веб-страницы.

  • Основные части программного кода должны быть разбиты на подпрограммы - модули.

  • Каждый модуль должен выполнять только свою часть работы.

  • Все программируемые части проекта должны быть реализованы на языке программирования JavaScript.

  • Верстка должна быть написана на HTML/CSS без использования фреймворков.

    2. Требования к программной части проекта:
    Основную логику веб-приложения должны осуществлять два модуля:

  • Модуль загрузки JSON-файла по ссылке,десериализация его в js-объект, проверка на корректность.

  • Модуль рисования - рисует в canvas UML Class диаграмму по js-объекту.

    3. Алгоритм работы программы:
    Входным параметром является ссылка, ведущая к json-файлу. Программа работает по следующему алгоритму:

  • Выдать ошибку в случае пустой строки

  • Выдать ошибку, если не удалось загрузить json по ссылке

  • Выдать ошибку, если не удалось выполнить восстановление json объекта из последовательности байт

  • Выдать ошибку, если json является неполным

  • Отрисовать uml-class диаграмму




    3. Дизайн:

  • Веб-приложение будет состоять из одной страницы с динамической подгрузкой конента для визуализации UML-диаграмм. Изначально на странице будет расположен input для ввода ссылки на JSON-файл, а также кнопка, запускающая процесс визуализации.




  • После нажатия на кнопку чуть ниже всех элементов будет располагаться Canvas-элемент на котором будет обрисовывается UML Class диаграмма.
